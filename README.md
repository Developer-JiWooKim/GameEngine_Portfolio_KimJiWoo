# GameEngine_Portfolio_KimJiWoo
GameEngine Portfolio Project

포트폴리오 관련 메모

1 Unity 6 씬 구성 - 실행 가능한 Unity 6 씬을 구성합니다. 씬 안에는 시작 위치, 주요 공간, 확인 가능한 오브젝트가 포함되어야 합니다. --> 완료

2 GameObject와 Component 구조 정리 - 프로젝트의 주요 오브젝트를 Hierarchy 안에서 의도에 맞게 배치합니다. 각 오브젝트에는 역할에 맞는 Component를 적용하고, 이름만으로도 대략적인 용도를 파악할 수 있도록 정리합니다. --> 완료

3 레벨 또는 플레이 공간 제작 - 간단한 레벨, 방, 통로, 필드, 훈련장, 퍼즐 공간 중 하나 이상을 제작합니다. 공간은 단순 배치가 아니라 사용자가 장면의 구조를 이해할 수 있는 형태여야 합니다.
	이미 만들어진 미로 생성 활용 - 미로 두개를 생성해서 Tab으로 교체 가능하게, 원래 미로, 환영 미로 서로 다른 구조를 지닌 미로로 전환해가며 못지나가는 길을 미로를 교체해서 지나갈 수도 있게 --> 완료

4 카메라와 Game View 구성 - Main Camera를 배치하고 위치, 회전, 투영 방식, 시야 범위를 조정하여 주요 공간과 오브젝트가 Game View에서 확인되도록 구성합니다. --> 완료

5 조명과 머티리얼 적용 - 조명과 머티리얼을 사용하여 장면의 분위기, 구역 구분, 오브젝트 성격 중 하나 이상을 표현합니다. 기본 오브젝트만 배치한 상태로 제출하지 않습니다. --> 쉐이더 그래프를 이용해서 미로의 머테리얼을 더 공포스럽게 연출
	미로 교체될 때 화면 일렁임? 환영미로? 일때는 약간 신비한 분위기 연출을 조명과 머티리얼 활용? --> 추가하면 좋을듯

6 리소스와 에셋 관리 - 프로젝트에 사용한 리소스, 프리팹, 머티리얼, 사운드, 이미지, VFX 파일 등을 폴더 구조 안에서 정리합니다. 외부 에셋을 사용할 경우 프로젝트 안에서 사용 위치를 확인할 수 있어야 합니다.
	미로들 외부 에셋 사용 할듯? Assets/External
		--> 완료, 내가 만든 에셋은  MyAssets에서 관리하고 나머지 외부 에셋들을 따로 건들지 않음

7 이동 또는 경로 시스템 적용 - CharacterController, NavMeshAgent, NavMeshSurface 중 하나 이상을 사용해 캐릭터 이동, 목표 지점 이동, 이동 가능 영역 확인 중 하나를 구현합니다. --> 완료
		캐릭터 이동 --> CharacterController 기반으로 움직이게 설정
		플레이어 입력 --> PlayerInput 유니티 내장 컴포넌트 사용으로 입력 받도록 설정
		몬스터 추적 AI --> NavMeshAgent, 미로 생성시 런타임에 Bake해서 NavMeshSurface 로 저장하고 미로가 변경될때마다 해당 NavMeshSurface를 따라 추적할 수 있게 설정

8 엔진 기능 적용 - 다음 항목 중 최소 2개 이상을 실제 기능으로 적용합니다.
	Animation 또는 Animator --> 유령? 혹은 몬스터? 외부 에셋으로 교체하면 적용
	UI --> 현재도 사용중이긴 한데 더 꾸밀듯
	사운드 --> 사운드 - 추가 예정(BGM, UI클릭 사운드), 공포 분위기를 컨셉으로 잡고 몬스터가 플레이어발견하면 비명? 효과음, 브금은 스산한걸로, 발견되면 점점고조되는 추격브금
	VFX --> 추가 예정?, 플레이어가 미로를 전환할때 브금, 화면 일렁임? 연출 동안 게임 스탑, 전환후 다시 게임 플레이
	물리 Trigger --> Goal Point, MonsterAttackRange에서 사용중, 추가할거면 아이템이나 파밍, 상호작용(클라이언트), 맵의 5개 정도 랜덤한 위치에 열쇠? 같은 오브젝트 두고 (UI에 현재 모은 개수 표시) 전부 획득해야 탈출할 수 있게 룰 추가 그에 따른 UI와 에셋 추가 예정		

9 실행 결과 확인 - 프로젝트를 실행했을 때 구현된 장면과 기능을 확인할 수 있어야 합니다. 단순 파일 제출이나 미완성 씬만으로는 제출 기준을 충족하지 않습니다. --> 프로젝트 완성 시 자동으로 완료됨
